Config

Get Data

Configuration data can be stored across multiple files in a hierarchical fashion where each new "layer" modifies the existing layer. It may be difficult to ascertain what the final result looks like. Use this call to get the final result.

Get Entity

Entity data can be stored across multiple files in a hierarchical fashion where each new "layer" modifies the existing layer. It may be difficult to ascertain what the final result looks like. Use this call to get the final result.

Get Search

Returns information about how the search engine (elasticsearch) is constructed. There should never be any reason to call this.

Reload

Config

Reloads the data configuration file for the specified client. Use this whenever you make a change to any data configuration files related to this client.

Segments

Reloads all segments for the specified client. Use when you want to force all segments to be updated for the client. Should seldom be called.

All Segments

Reloads all segments for ALL clients. Should almost NEVER be called.

Search

Rebuild Index

Rebuilds the search index. If for some reason search results are not working the way you expect, it is possible that the search index may not be completely up to date. Call this to force the index to match the data we have.

Reindex Index

This should only be used if something changes in the search configuration file. This quickly rebuilds the search index using data from the existing index as opposed to reading it from the database. Should seldom be called.

Sync Index

Verifies that all internal data is synced with the search index. This should happend automatically so it is unlikely that this should ever be called.

Statistics

Returns search statistics for the search engine (elasticsearch). Should seldom be called.
